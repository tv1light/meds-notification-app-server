## 1. Авторизация

### Регистрация

**POST** `/auth/register`

**Тело запроса:**
```json
{
  "login": "ivan123",
  "password": "secret99"
}
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| login | string | да | 4–12 символов |
| password | string | да | 8–14 символов |

**Ответ 201 Created:**
```json
{
  "id": 1,
  "login": "ivan123"
}
```

**Ошибки:**
- `400` — логин/пароль не прошли валидацию
- `409` — логин уже занят

---

### Вход

**POST** `/auth/login`

**Тело запроса:**
```json
{
  "login": "ivan123",
  "password": "secret99"
}
```

**Ответ 200 OK:**
```json
{
  "id": 1,
  "login": "ivan123"
}
```

**Ошибки:**
- `401` — неверный логин или пароль

> **Важно:** сессии и токены не используются. `id` пользователя из ответа нужно сохранить на клиенте — он используется во всех запросах где требуется `userId`.

## 2. Справочник стран

### Получить все страны

**GET** `/countries`

**Ответ 200 OK:**
```json
[
  { "id": 1, "name": "Россия" },
  { "id": 2, "name": "Германия" }
]
```

---

### Получить страну по ID

**GET** `/countries/{id}`

**Ответ 200 OK:**
```json
{ "id": 1, "name": "Россия" }
```

**Ошибки:** `404` — страна не найдена

---

### Создать страну

**POST** `/countries`

**Тело запроса:**
```json
{ "name": "Франция" }
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| name | string | да | 3–56 символов, уникальное |

**Ответ 201 Created:**
```json
{ "id": 3, "name": "Франция" }
```

**Ошибки:** `400` — валидация, `409` — уже существует

---

### Обновить страну

**PUT** `/countries/{id}`

**Тело запроса:**
```json
{ "name": "Франция (обновлено)" }
```

**Ответ 200 OK:**
```json
{ "id": 3, "name": "Франция (обновлено)" }
```

---

### Удалить страну

**DELETE** `/countries/{id}`

**Ответ 204 No Content**

**Ошибки:** `404` — страна не найдена

## 6. Препараты

### Получить все

**GET** `/medications?sort=asc`

| Параметр | Тип | Обязательно | Значения |
|----------|-----|-------------|----------|
| sort | string | нет | `asc` (по умолчанию), `desc` — сортировка по названию |

**Ответ 200 OK:**
```json
[
  {
    "id": 1,
    "tradeName": "Ибупрофен",
    "activeSubstanceId": 1,
    "activeSubstanceName": "Ibuprofen",
    "releaseFormId": 1,
    "releaseFormName": "Таблетки",
    "manufacturerId": 1,
    "manufacturerName": "ООО БиоФарм",
    "manufacturerCountry": "Россия",
    "dosageValue": 200.000,
    "dosageUnit": "MG"
  }
]
```

---

### Получить по ID

**GET** `/medications/{id}`

---

### Создать

**POST** `/medications`

```json
{
  "tradeName": "Ибупрофен",
  "activeSubstanceId": 1,
  "releaseFormId": 1,
  "manufacturerId": 1,
  "dosageValue": 200.000,
  "dosageUnit": "MG"
}
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| tradeName | string | да | 2–50 символов |
| activeSubstanceId | number | да | ID из справочника |
| releaseFormId | number | да | ID из справочника |
| manufacturerId | number | да | ID из справочника |
| dosageValue | number | да | 0.001–100000.000 |
| dosageUnit | string | да | см. [единицы дозировки](#единицы-дозировки-dosageunit) |

**Ответ 201 Created** — объект препарата (см. выше)

---

### Обновить

**PUT** `/medications/{id}`

Тело — то же что при создании.

---

### Удалить

**DELETE** `/medications/{id}`

**Ответ 204 No Content**

## 7. Курсы терапии

Курс всегда принадлежит конкретному пользователю.

### Получить курсы пользователя

**GET** `/courses?userId=1&sort=asc`

| Параметр | Тип | Обязательно | Значения |
|----------|-----|-------------|----------|
| userId | number | да | ID пользователя |
| sort | string | нет | `asc` (по умолчанию), `desc` — по дате начала |

**Ответ 200 OK:**
```json
[
  {
    "id": 1,
    "userId": 1,
    "userLogin": "ivan123",
    "medicationId": 1,
    "medicationName": "Ибупрофен",
    "startDate": "2025-06-01",
    "endDate": "2025-06-14",
    "notes": "Принимать после еды",
    "createdAt": "2025-05-30T10:00:00+03:00"
  }
]
```

---

### Получить курс по ID

**GET** `/courses/{id}`

---

### Создать курс

**POST** `/courses`

```json
{
  "userId": 1,
  "medicationId": 1,
  "startDate": "2025-06-01",
  "endDate": "2025-06-14",
  "notes": "Принимать после еды"
}
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| userId | number | да | ID существующего пользователя |
| medicationId | number | да | ID существующего препарата |
| startDate | string | да | Формат `YYYY-MM-DD` |
| endDate | string | нет | Формат `YYYY-MM-DD`, не раньше startDate. Если не указан — курс бессрочный |
| notes | string | нет | Произвольный текст |

**Ответ 201 Created** — объект курса

---

### Обновить курс

**PUT** `/courses/{id}`

Тело — то же что при создании (кроме `userId` — пользователя менять нельзя).

---

### Удалить курс

**DELETE** `/courses/{id}`

**Ответ 204 No Content**

> При удалении курса каскадно удаляются все связанные расписания, напоминания и записи журнала.

## 8. Расписания приёма

Расписание задаёт время напоминания внутри курса. На один курс можно создать несколько расписаний (например, 08:00 и 20:00).

### Получить расписания курса

**GET** `/schedules?courseId=1`

| Параметр | Тип | Обязательно |
|----------|-----|-------------|
| courseId | number | да |

**Ответ 200 OK:**
```json
[
  {
    "id": 1,
    "courseId": 1,
    "medicationName": "Ибупрофен",
    "remindTime": "08:00:00",
    "createdAt": "2025-05-30T10:05:00+03:00"
  },
  {
    "id": 2,
    "courseId": 1,
    "medicationName": "Ибупрофен",
    "remindTime": "20:00:00",
    "createdAt": "2025-05-30T10:05:00+03:00"
  }
]
```

---

### Получить по ID

**GET** `/schedules/{id}`

---

### Создать расписание

**POST** `/schedules`

```json
{
  "courseId": 1,
  "remindTime": "08:00:00"
}
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| courseId | number | да | ID существующего курса |
| remindTime | string | да | Формат `HH:mm:ss` |

**Ответ 201 Created** — объект расписания

---

### Обновить время

**PUT** `/schedules/{id}`

```json
{
  "courseId": 1,
  "remindTime": "09:00:00"
}
```

---

### Удалить расписание

**DELETE** `/schedules/{id}`

**Ответ 204 No Content**

---

## 9. Напоминания

Напоминание — конкретный момент времени (дата + время), сгенерированный из расписания.

### Получить напоминания расписания

**GET** `/reminders?scheduleId=1`

| Параметр | Тип | Обязательно |
|----------|-----|-------------|
| scheduleId | number | да |

**Ответ 200 OK:**
```json
[
  {
    "id": 1,
    "scheduleId": 1,
    "courseId": 1,
    "medicationName": "Ибупрофен",
    "remindAt": "2025-06-01T08:00:00+03:00",
    "status": "PENDING"
  }
]
```

---

### Получить по ID

**GET** `/reminders/{id}`

---

### Создать напоминание

**POST** `/reminders`

```json
{
  "scheduleId": 1,
  "remindAt": "2025-06-01T08:00:00+03:00"
}
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| scheduleId | number | да | ID существующего расписания |
| remindAt | string | да | ISO 8601 с часовым поясом |

**Ответ 201 Created** — объект напоминания со статусом `PENDING`

---

### Изменить статус напоминания

**PATCH** `/reminders/{id}/status?value=DONE`

| Параметр | Тип | Обязательно | Значения |
|----------|-----|-------------|----------|
| value | string | да | `PENDING`, `DONE`, `SKIPPED`, `SNOOZED` |

**Ответ 200 OK** — обновлённый объект напоминания

---

### Удалить напоминание

**DELETE** `/reminders/{id}`

**Ответ 204 No Content**

## 10. Журнал приёмов

Фиксирует факт того, что пользователь сделал с напоминанием: принял, пропустил или перенёс.

### Получить журнал пользователя

**GET** `/intakes?userId=1&sort=desc`

| Параметр | Тип | Обязательно | Значения |
|----------|-----|-------------|----------|
| userId | number | да | ID пользователя |
| sort | string | нет | `desc` (по умолчанию), `asc` — по времени действия |

**Ответ 200 OK:**
```json
[
  {
    "id": 1,
    "reminderId": 1,
    "userId": 1,
    "medicationName": "Ибупрофен",
    "action": "DONE",
    "actionedAt": "2025-06-01T08:03:00+03:00",
    "snoozedTo": null
  },
  {
    "id": 2,
    "reminderId": 2,
    "userId": 1,
    "medicationName": "Ибупрофен",
    "action": "SNOOZED",
    "actionedAt": "2025-06-02T20:01:00+03:00",
    "snoozedTo": "2025-06-02T21:00:00+03:00"
  }
]
```

---

### Получить запись по ID

**GET** `/intakes/{id}`

---

### Зафиксировать приём / пропуск / перенос

**POST** `/intakes`

```json
{
  "reminderId": 1,
  "userId": 1,
  "action": "DONE",
  "snoozedTo": null
}
```

| Поле | Тип | Обязательно | Ограничения |
|------|-----|-------------|-------------|
| reminderId | number | да | ID существующего напоминания |
| userId | number | да | ID пользователя |
| action | string | да | `DONE`, `SKIPPED`, `SNOOZED` |
| snoozedTo | string | только при `SNOOZED` | ISO 8601 с часовым поясом |

**Ответ 201 Created** — объект записи журнала

> При создании записи статус соответствующего напоминания обновляется автоматически.

## 11. Коды ошибок

Все ошибки возвращаются в едином формате:

```json
{
  "message": "Препарат не найден"
}
```

| Код | Когда |
|-----|-------|
| 400 Bad Request | Ошибка валидации входных данных |
| 401 Unauthorized | Неверный логин или пароль |
| 404 Not Found | Запрошенный объект не найден |
| 409 Conflict | Объект с таким именем уже существует |
| 500 Internal Server Error | Внутренняя ошибка сервера |

## 12. Справочник значений

### Единицы дозировки (dosageUnit)

| Значение | Расшифровка |
|----------|-------------|
| `MCG` | Микрограммы (мкг) |
| `MG` | Миллиграммы (мг) |
| `G` | Граммы (г) |
| `ML` | Миллилитры (мл) |
| `MCU` | Микромеждународные единицы (мкМЕ) |
| `IU` | Международные единицы (МЕ) |
| `DOSE` | Доза/нажатие (для спреев и ингаляторов) |

### Статусы напоминания (ReminderStatus)

| Значение | Расшифровка |
|----------|-------------|
| `PENDING` | Ожидает — напоминание ещё не наступило или не обработано |
| `DONE` | Подтверждён — пользователь принял лекарство |
| `SKIPPED` | Пропущен |
| `SNOOZED` | Перенесён — отложен на другое время |

### Действия с приёмом (IntakeAction)

| Значение | Расшифровка |
|----------|-------------|
| `DONE` | Принял |
| `SKIPPED` | Пропустил |
| `SNOOZED` | Перенёс |
